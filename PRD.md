# PRD：小学生英语单词背诵应用（PWA）

## 项目信息

- **Language**: 中文
- **技术栈**: HTML5 + CSS + JavaScript（PWA）
- **项目名称**: vocab_app
- **原始需求**: 开发一款面向小学生的英语单词背诵安卓应用，基于译林版英语教材词库，支持多种背诵模式与艾宾浩斯智能复习，纯本地离线可用，无登录无广告。

---

## 产品目标

1. **高效记忆**：通过多种背诵模式和艾宾浩斯遗忘曲线智能复习，帮助小学生高效掌握英语单词。
2. **零门槛使用**：纯本地PWA应用，无需注册登录、无需联网，打开即用，降低使用门槛。
3. **儿童友好体验**：卡通活泼的UI设计与TTS发音辅助，让单词学习过程轻松有趣。

---

## 用户故事

1. **作为一名小学生**，我想通过卡片翻面模式背诵单词，这样我可以先看英文猜中文，再翻转验证，加深记忆。
2. **作为一名小学生**，我想通过选择题模式复习单词，这样我可以在多个选项中选出正确释义，检验我的掌握程度。
3. **作为一名小学生**，我想通过拼写模式巩固单词，这样我可以通过手动拼写加深对单词拼写的记忆。
4. **作为一名小学生**，我想在快要遗忘某个单词时收到复习提醒，这样我可以按科学节奏复习，不用自己记着什么时候该复习。
5. **作为家长/学生**，我想手动添加学校要求的额外单词到词库中，这样我可以灵活地管理需要学习的单词。

## 功能说明

### 1. 内置词库

- 预置译林版英语教材3-6年级全部单词，数据结构包含：英文单词、音标、词性、中文释义、例句、所属年级/单元。
- 词库以JSON格式内嵌于应用中，离线可用。
- 支持与自定义词库统一检索和浏览。

### 2. 卡片翻面背诵

- 正面显示英文单词，用户思考后点击/滑动翻转至背面查看中文释义。
- 每张卡片提供"已掌握"和"再复习"两个操作按钮。
- 选择"已掌握"的单词在当前轮次中不再出现；选择"再复习"的单词在当前轮次末尾再次出现。

### 3. 选择题背诵

- 题目展示英文单词，下方呈现4个中文释义选项（1正确+3干扰项）。
- 干扰项从同单元/同年级的其他单词中随机抽取。
- 选择后即时反馈：正确显示绿色对勾，错误显示红色叉号并标出正确答案。
- 答错的单词自动加入复习队列。

### 4. 拼写背诵

- 展示中文释义，用户在输入框中拼写英文单词。
- 支持实时字符匹配提示（如已输入部分正确字母高亮）。
- 提交后判断对错，错误时显示正确拼写并标出差异字母。
- 可选提供"提示"按钮，显示首字母或单词长度。

### 5. 艾宾浩斯智能复习

- 遗忘曲线复习间隔参考值：第1次复习=1天后，第2次=2天后，第3次=4天后，第4次=7天后，第5次=15天后，第6次=30天后。
- 每个单词首次学习后启动复习计时。
- 用户完成一次复习后，系统根据当前复习轮次自动计算下次复习时间。
- 首页展示"今日待复习"入口，显示需复习的单词数量。

### 6. TTS发音

- 使用浏览器原生Web Speech API实现单词和例句朗读。
- 点击单词旁的喇叭图标即可播放发音。
- 发音语言设置为英语（en-US或en-GB）。

### 7. 学习进度记录

- 使用IndexedDB持久化存储学习数据。
- 每个单词记录：学习状态（new/learning/mastered）、首次学习时间、上次复习时间、下次应复习时间、复习轮次、在当前模式下的正确/错误次数。
- 无打卡功能，无统计图表，专注单词本身的状态追踪。

### 8. PWA离线支持

- 注册Service Worker，缓存应用Shell（HTML/CSS/JS）及词库JSON。
- 使用Cache API进行资源缓存策略：静态资源使用Cache First，动态数据使用Network First + Fallback。
- 生成manifest.json，支持添加到主屏幕。
- TTS功能在网络可用时体验更佳，但离线时其他功能不受影响。

### 9. 卡通风格UI

- 主色调：明亮活泼的配色（如橙色、绿色、蓝色为主色）。
- 图标和插画采用卡通风格，可使用开源卡通图标库。
- 字体选用圆润可爱的字体（如站酷快乐体等免费商用字体）。
- 按钮大而清晰，方便儿童触控操作。
- 动画适度：翻转卡片动画、答对庆祝动画（如星星飘落），但不过度以免分散注意力。

## 待确认问题

1. **TTS离线方案**：Web Speech API在部分安卓设备/浏览器上离线可能不可用，是否需要预置音频文件作为fallback？这会显著增加应用体积。
2. **词库数据来源**：译林版3-6年级单词的精确列表及例句，需确认是否有版权问题，或是否需要自行编写例句。
3. **APK打包方案**：确认使用TWA（Trusted Web Activity）还是Capacitor/Cordova方案？TWA更轻量但要求PWA通过可安装性检测；Capacitor可调用更多原生能力。
4. **遗忘曲线参数**：当前使用固定间隔序列，是否需要根据用户实际记忆表现动态调整间隔（如SM-2算法）？
5. **自定义词库上限**：是否需要对自定义词库的单词数量设置上限，避免IndexedDB存储过大？
6. **多子女使用场景**：是否需要在同一设备上支持多个孩子的学习进度切换？（当前设计为单用户本地应用）
